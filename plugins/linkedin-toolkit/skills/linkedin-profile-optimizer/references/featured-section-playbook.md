# Featured Section Playbook (Software-Engineer Job Seekers)

Featured = only profile area user controls above-the-fold proof order. For engineer job seekers, make it a recruiter-facing portfolio of **engineering artifacts**, not LinkedIn content highlights.

## Quality over quantity

**3 strong items is a heuristic, not a hard rule.** Two strong items beat three filler. One strong item beats weak slots.

Rotate quarterly or before focused application push.

## Content types allowed

- External links (GitHub, personal site, project / portfolio pages)
- PDFs (architecture writeups, technical case studies, sanitized design docs)
- Posts (own LinkedIn posts; optional)
- Articles (LinkedIn native or external technical writing)
- Presentations (talk decks)
- Videos (talk recordings, demos, walkthroughs)

## Custom thumbnails

- **Dimensions:** 1200 x 627 px
- Use **outcome-driven titles**, not generic labels
- ❌ "My blog post about engineering" → ✅ "Cutting p95 latency 68% on a 3M-doc retrieval pipeline"
- ❌ "Case study PDF" → ✅ "Rebuilding our checkout: architecture writeup + 23% conversion lift"
- ❌ "My GitHub" → ✅ "Selected OSS work · LLM evals, distributed tracing, Go libraries"
- High contrast, bold sans-serif text, minimal decoration

## Featured triplet for engineers

Default order: engineering artifacts first, content marketing last.

### Slot 1 (always): GitHub / portfolio / personal site

Code or shipped work, one click from profile.

Strongest:

- Curated GitHub profile or pinned repos matching target role family
- Personal site with selected projects: problem → approach → outcome → tech
- Frontend / design-adjacent: portfolio with live demos + case studies

Name slot by **outcome produced**, not artifact name. "Selected backend work · last 12 months" beats "My GitHub".

### Slot 2 (recommended): Technical writeup or signature project

One piece recruiter should deep-read:

- Public technical writeup (architecture case study, postmortem, RFC, design doc)
- Signature OSS contribution or meaningful upstream PR
- Published talk recording / conference deck
- Sanitized shipped-work artifact (e.g., public API / developer-facing product owned)

Prove **depth**: how user thinks, not just what shipped.

### Slot 3 (optional): Top LinkedIn post, talk, or secondary project

Use only if signal is real. SWE value order:

1. Conference talk recording
2. Second signature project / OSS contribution
3. Technical blog post
4. Top LinkedIn post **only if** it shows engineering judgment and isn't generic career advice

**LinkedIn posts are explicitly de-prioritized.** Don't manufacture content to fill slot. If no strong Slot 3, leave empty.

## Rotation schedule

- **Quarterly audit:** replace items older than 12 weeks OR mismatched to target role
- **Before focused application push:** align Featured with role family + seniority
- **After publishing something new:** swap if stronger than current Featured

## Anti-patterns

- Current employer's corporate blog post when user's own writeup exists
- Generic "About me" PDFs
- Dated webinar recordings
- Random "selected writing" with no through-line
- Default LinkedIn-suggested content from connection posts
- "Book a call" landing pages or service pitches
- LinkedIn job-seeking posts crowding out engineering signal
