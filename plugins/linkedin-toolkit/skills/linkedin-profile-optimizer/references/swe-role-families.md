# SWE Role Families — Quick Reference

Per role family: headline keywords, what to emphasize in Experience, and what makes the strongest Featured artifacts. Combine with the JD-keyword workflow — the role family sets defaults, the JDs refine them for the specific roles you're targeting.

## Backend

- **Headline keywords:** distributed systems, services, throughput, latency, [language: Go / Python / Java / Rust], [datastore: Postgres / Kafka / Redis], multi-tenant, microservices
- **Experience emphasis:** system design decisions, throughput/latency wins, reliability/SLO work, refactors at scale, on-call leadership
- **Best Featured artifacts:** architecture writeups, p99-reduction case studies, OSS server-side libraries, RFCs you authored

## Frontend

- **Headline keywords:** web, React / Vue / Svelte, performance, accessibility, design systems, TypeScript, component libraries
- **Experience emphasis:** perf wins (LCP, CLS, INP), a11y improvements, design-system shipping, conversion / engagement lift, framework migrations
- **Best Featured artifacts:** live demos, perf-audit writeups, public design-system contributions, framework migration case studies

## Full-stack

- **Headline keywords:** end-to-end, product engineering, full-stack, [stack: TypeScript + Postgres + React, etc.], zero-to-one
- **Experience emphasis:** zero-to-one feature shipping, breadth across layers with depth in one, ownership scope, cross-layer optimizations
- **Best Featured artifacts:** shipped products you can demo, OSS contributions spanning frontend + backend, end-to-end case studies

## Platform / DevOps / SRE

- **Headline keywords:** infrastructure, reliability, observability, IaC, Kubernetes, Terraform, CI/CD, SRE, platform engineering
- **Experience emphasis:** SLO/SLA wins, deploy velocity, infra cost reductions, incident reduction, internal developer experience
- **Best Featured artifacts:** public postmortems / write-ups, IaC modules / templates, OSS infra tools, talk recordings on platform topics

## AI / ML / LLM

- **Headline keywords:** ML, LLM, RAG, agents, evals, [frameworks: PyTorch / JAX / LangChain / LangGraph], embeddings, fine-tuning, inference
- **Experience emphasis:** eval frameworks built, dataset work, model deployment scale, latency/cost wins on inference, production ML reliability
- **Best Featured artifacts:** published evals, model cards, OSS LLM / agent contributions, papers, demo videos of agentic systems

## Security

- **Headline keywords:** AppSec, security engineering, threat modeling, vulnerability management, compliance, SOC2 / ISO27001, CISSP / OSCP
- **Experience emphasis:** vuln reduction, audit findings closed, security architecture decisions, compliance program work, incident response
- **Best Featured artifacts:** CVEs found, conference talks, security writeups, OSS security tools, sanitized threat-modeling docs

## Staff / Principal IC

- **Headline keywords:** Staff Engineer / Principal Engineer, technical leadership, architecture, cross-team, mentorship, [domain depth]
- **Experience emphasis:** org-wide impact, RFC / design-doc influence, multi-team initiatives, technical strategy, mentorship at scale (not headcount)
- **Best Featured artifacts:** architecture RFCs, conference talks, design docs (sanitized), technical strategy writeups, OSS leadership

## Engineering Manager

- **Headline keywords:** Engineering Manager, team scaling, hiring, delivery, [domain: backend / platform / etc.], engineering leadership
- **Experience emphasis:** team building (size + retention), hiring/onboarding work, delivery cadence, shipping outcomes attributed to team, performance management
- **Best Featured artifacts:** hiring / retention writeups, team org docs (sanitized), retros / postmortems you ran, talks on engineering leadership

## Cross-family notes

- **Stack overlap is normal.** A backend engineer who also runs infra on a small team can fit either Backend or Platform — pick by **target role**, not current title.
- **Staff IC vs EM is a hard pivot.** If pivoting, lead with the destination role's keywords and use Featured for proof artifacts that match the *destination*, not the origin. See `search-modes.md` → Mode 3.
- **AI/ML and Backend overlap heavily** in 2026 for production-LLM roles. If your target JDs use "ML platform" or "ML engineer (production)", you may need keywords from both families.
