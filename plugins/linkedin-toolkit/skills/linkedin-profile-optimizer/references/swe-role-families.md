# SWE Role Families — Quick Reference

Per family: headline keywords, Experience emphasis, strongest Featured artifacts. Use JD-keyword workflow first when JDs exist; role family gives defaults, JDs refine.

## Backend

- **Headline keywords:** distributed systems, services, throughput, latency, [language: Go / Python / Java / Rust], [datastore: Postgres / Kafka / Redis], multi-tenant, microservices
- **Experience emphasis:** system design, throughput/latency wins, reliability/SLO, scale refactors, on-call leadership
- **Best Featured artifacts:** architecture writeups, p99 case studies, OSS server-side libraries, RFCs

## Frontend

- **Headline keywords:** web, React / Vue / Svelte, performance, accessibility, design systems, TypeScript, component libraries
- **Experience emphasis:** LCP/CLS/INP wins, a11y, design systems, conversion / engagement lift, framework migrations
- **Best Featured artifacts:** live demos, perf-audit writeups, public design-system work, migration case studies

## Full-stack

- **Headline keywords:** end-to-end, product engineering, full-stack, [stack: TypeScript + Postgres + React, etc.], zero-to-one
- **Experience emphasis:** zero-to-one shipping, breadth across layers + depth in one, ownership scope, cross-layer optimizations
- **Best Featured artifacts:** demoable products, OSS spanning frontend + backend, end-to-end case studies

## Platform / DevOps / SRE

- **Headline keywords:** infrastructure, reliability, observability, IaC, Kubernetes, Terraform, CI/CD, SRE, platform engineering
- **Experience emphasis:** SLO/SLA wins, deploy velocity, infra cost cuts, incident reduction, developer experience
- **Best Featured artifacts:** public postmortems / writeups, IaC modules / templates, OSS infra tools, platform talks

## AI / ML / LLM

- **Headline keywords:** ML, LLM, RAG, agents, evals, [frameworks: PyTorch / JAX / LangChain / LangGraph], embeddings, fine-tuning, inference
- **Experience emphasis:** eval frameworks, datasets, model deployment scale, inference latency/cost, production ML reliability
- **Best Featured artifacts:** published evals, model cards, OSS LLM / agent work, papers, agentic-system demos

## Security

- **Headline keywords:** AppSec, security engineering, threat modeling, vulnerability management, compliance, SOC2 / ISO27001, CISSP / OSCP
- **Experience emphasis:** vuln reduction, audit findings closed, security architecture, compliance programs, incident response
- **Best Featured artifacts:** CVEs, conference talks, security writeups, OSS security tools, sanitized threat models

## Staff / Principal IC

- **Headline keywords:** Staff Engineer / Principal Engineer, technical leadership, architecture, cross-team, mentorship, [domain depth]
- **Experience emphasis:** org-wide impact, RFC / design-doc influence, multi-team initiatives, technical strategy, mentorship at scale (not headcount)
- **Best Featured artifacts:** architecture RFCs, conference talks, sanitized design docs, technical strategy writeups, OSS leadership

## Engineering Manager

- **Headline keywords:** Engineering Manager, team scaling, hiring, delivery, [domain: backend / platform / etc.], engineering leadership
- **Experience emphasis:** team building (size + retention), hiring/onboarding, delivery cadence, team shipping outcomes, performance management
- **Best Featured artifacts:** hiring / retention writeups, sanitized org docs, retros / postmortems, engineering-leadership talks

## Cross-family notes

- **Stack overlap is normal.** Pick family by **target role**, not current title.
- **Staff IC vs EM is a hard pivot.** Lead with destination keywords; use Featured proof matching *destination*, not origin. See `search-modes.md` → Mode 3.
- **AI/ML and Backend overlap heavily** in 2026 production-LLM roles. For "ML platform" or "ML engineer (production)", mix both families.
