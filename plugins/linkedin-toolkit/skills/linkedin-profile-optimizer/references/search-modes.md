# Search Modes

How you optimize a job-seeker profile depends heavily on **how visible your search is allowed to be**. A profile change that's right for an "open search" candidate will get a discreetly-searching engineer fired.

Pick one mode at intake and apply its constraints across every section.

## Mode 1: Open search

You're publicly looking for work. Current employer (if any) knows, or you don't have a current employer.

| Surface | Setting |
| --- | --- |
| Open to Work badge | **On** (visible to all) |
| Headline | Lead with **target role**; "Open to [Role] roles" is fine if not crowding more important keywords |
| About | Direct: state target role, what you're looking for, contact |
| Banner | Target-role tagline + optional "Open to [Role] roles" line |
| Featured | Lead with portfolio + best technical writeup |
| Recruiter-facing | Email and/or LinkedIn DM both visible; explicit availability |

## Mode 2: Discreet search

You're employed and don't want your current employer, manager, or coworkers to know you're looking.

| Surface | Setting |
| --- | --- |
| Open to Work badge | **Recruiters-only mode** (LinkedIn Settings → Job seeking preferences → "Share with recruiters only") |
| Headline | Current role + target-role keywords blended naturally — no "Open to X" |
| About | Subtle target-role positioning via specialties and what-I-care-about; **don't say "I'm looking for"** |
| Banner | Neutral target-role tagline; no Open-to-Work language |
| Featured | Same as open search; engineering artifacts don't signal job-seeking |
| Recruiter-facing | Contact info via email only (DMs from your network may include current colleagues) |

**Critical discreet-search hygiene:**

- **Disable activity broadcasts** before editing: Settings → Account preferences → "Share profile updates with network" → Off
- Make all profile edits in one batched session so partial changes don't leak
- Don't post about job-seeking topics; don't engage publicly with recruiter content
- Don't add new "Open to Work" or career-related sections that weren't there before — adds drama to your feed

## Mode 3: Career pivot

You're targeting a role family that differs from your current/recent one (e.g., backend engineer → ML engineer, IC → EM, EM → staff IC).

| Surface | Setting |
| --- | --- |
| Open to Work badge | On (or recruiters-only if also discreet) |
| Headline | **Lead with target role**, NOT current — e.g., "ML Engineer (transitioning from Backend) | LLM Eval, RAG | Python, PyTorch" |
| About | Explain the pivot in 1–2 sentences: transferable skills + concrete pivot proof (projects, courses, OSS, side work) |
| Banner | Target-role tagline; portfolio link showing pivot work |
| Featured | **Pivot proof first** — the project/repo/writeup that proves you can do the target work, not your strongest past-role artifact |
| Recruiter-facing | Direct about pivot; specify what target role + adjacent flexibility you'll consider |

## Mode 4: Career break / post-layoff

You're not currently employed (break, sabbatical, or post-layoff) and openly searching.

| Surface | Setting |
| --- | --- |
| Open to Work badge | **On** (visible to all) |
| Headline | Target-role + skills; the break itself doesn't need headline real estate |
| About | One honest sentence on the break/layoff is fine if context helps; lead with target role and what you bring |
| Banner | Target-role tagline; "Available immediately" if true |
| Featured | Portfolio + signature work; if break was used productively (OSS, learning, side projects), feature one of those |
| Recruiter-facing | Contact + availability ("available immediately" / start date) |

Post-layoff specifically: LinkedIn now has a "former employees" sidebar near company pages; recruiters often source there proactively. Make sure your past role's company is set correctly so you appear in that pool.

## Mode 5: Employed-but-exploring

You're employed, not actively searching, but open to a great offer for the right role.

| Surface | Setting |
| --- | --- |
| Open to Work badge | **Off** or recruiters-only |
| Headline | Current role + target-role keywords, light-touch |
| About | Not framed as job-seeking; signals interest in adjacent/aspirational work via specialties and "what I care about" |
| Banner | Neutral / current-role |
| Featured | Strongest work — quality over volume; signals you're worth poaching |
| Recruiter-facing | Selective: contact info present but no urgency cues |

Use this mode when the goal is **inbound recruiter interest for the right offer**, not active applications.

## Cross-mode rule

Whichever mode you pick, the **target-role keywords** still need to be present in headline + About + Skills + Experience. The mode controls *tone and visibility*, not whether you optimize for the target role at all.
