# Search Modes

Pick one mode at intake. Apply its constraints across every section. Mode controls tone + visibility; target-role keywords still belong in headline, About, Skills, Experience.

## Mode 1: Open search

Publicly looking. Employer knows, or no current employer.

| Surface | Setting |
| --- | --- |
| Open to Work badge | **On** (visible to all) |
| Headline | Lead with **target role**; "Open to [Role] roles" ok only if not crowding stronger keywords |
| About | Direct: target role, what user wants, contact |
| Banner | Target-role tagline + optional "Open to [Role] roles" |
| Featured | Portfolio + best technical writeup first |
| Recruiter-facing | Email and/or LinkedIn DM visible; explicit availability |

## Mode 2: Discreet search

Employed; avoid signaling search to employer / manager / coworkers.

| Surface | Setting |
| --- | --- |
| Open to Work badge | **Recruiters-only mode** (LinkedIn Settings → Job seeking preferences → "Share with recruiters only") |
| Headline | Current role + target-role keywords blended; no "Open to X" |
| About | Subtle target-role positioning through specialties + what-I-care-about; **don't say "I'm looking for"** |
| Banner | Neutral target-role tagline; no Open-to-Work language |
| Featured | Same as open search; engineering artifacts don't signal job-seeking |
| Recruiter-facing | Email only; DMs from network may include current colleagues |

**Critical discreet-search hygiene:**

- **Disable activity broadcasts** before editing: Settings → Account preferences → "Share profile updates with network" → Off
- Batch all profile edits in one session
- Don't post job-seeking content; don't engage publicly with recruiter content
- Don't add new "Open to Work" / career sections that weren't there before

## Mode 3: Career pivot

Target role family differs from current/recent one (e.g., backend → ML, IC → EM, EM → staff IC).

| Surface | Setting |
| --- | --- |
| Open to Work badge | On, or recruiters-only if also discreet |
| Headline | **Lead with target role**, NOT current — e.g., "ML Engineer (transitioning from Backend) | LLM Eval, RAG | Python, PyTorch" |
| About | Explain pivot in 1–2 sentences: transferable skills + proof (projects, courses, OSS, side work) |
| Banner | Target-role tagline; portfolio link showing pivot work |
| Featured | **Pivot proof first** — project/repo/writeup proving target work, not strongest origin artifact |
| Recruiter-facing | Direct about pivot; specify target role + adjacent flexibility |

## Mode 4: Career break / post-layoff

Not employed; openly searching after break, sabbatical, or layoff.

| Surface | Setting |
| --- | --- |
| Open to Work badge | **On** (visible to all) |
| Headline | Target role + skills; break doesn't need headline space |
| About | One honest break/layoff sentence only if useful; lead with target role + value |
| Banner | Target-role tagline; "Available immediately" if true |
| Featured | Portfolio + signature work; if break produced OSS / learning / side projects, feature one |
| Recruiter-facing | Contact + availability ("available immediately" / start date) |

Post-layoff: ensure past company is set correctly so user appears in LinkedIn's "former employees" sourcing pool near company pages.

## Mode 5: Employed-but-exploring

Employed, not actively applying, open to strong-fit offers.

| Surface | Setting |
| --- | --- |
| Open to Work badge | **Off** or recruiters-only |
| Headline | Current role + target-role keywords, light-touch |
| About | Not job-seeking framed; signal adjacent/aspirational interests via specialties + "what I care about" |
| Banner | Neutral / current-role |
| Featured | Strongest work; quality over volume; poaching signal |
| Recruiter-facing | Contact present, no urgency cues |

Use when goal is **inbound recruiter interest for the right offer**, not active applications.

## Cross-mode rule

Always include **target-role keywords** in headline + About + Skills + Experience. Mode controls tone/visibility, not optimization.
